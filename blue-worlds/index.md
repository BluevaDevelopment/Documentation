# Blue Worlds Documentation

Welcome to the official documentation for **Blue Worlds** - the advanced world management plugin for Minecraft servers.

## What is Blue Worlds?

Blue Worlds lets server owners and players create, manage, and customize Minecraft worlds through two independent systems:

- **Private Worlds**: player-owned personal worlds with full access control, flags, and backups.
- **Global Worlds**: server-wide administrator-managed worlds for lobbies, arenas, resource worlds, events, and more.

Both systems are built on a **template system** that copies pre-built worlds instead of generating terrain on demand, giving fast creation, low lag, and consistent results.

## Key Features

- **Private worlds** - every player can own multiple personal worlds.
- **Global worlds** - server-wide worlds for lobbies, arenas, events, etc.
- **Templates** - create worlds from reusable ZIP templates with flags and settings.
- **World reset** - reset private and global worlds back to their template.
- **Multi-dimension support** - overworld, nether and end linked dimensions.
- **70+ flags** - control PvP, building, mob spawning, weather, gamemode, fly, and more.
- **Access control** - trust, add, remove, kick, ban, tempban and whitelist players.
- **Java + Bedrock menus** - configurable GUIs for both platforms.
- **Backups** - automatic and manual backups for private and global worlds.
- **PlaceholderAPI / Floodgate / Geyser integration**.

## Requirements

- **Server**: Spigot/Paper 1.21.x or Paper 26.1+
- **Java**: 21 or higher
- **RAM**: 2-4 GB minimum (depending on number of worlds)

## Optional Dependencies

- **PlaceholderAPI** - For placeholder support in external plugins
- **ProtocolLib** - For enhanced protocol features
- **Floodgate / Geyser** - For Bedrock player support
